# SP2.2: Extended Format Support (DSF, MKA, WMA)

## Context

Lofty 0.23 handles 12 audio formats but doesn't support DSF, WMA, or MKA. The
original plan called for TagLib C++ FFI, but TagLib complicates musl static
linking for the NAS deployment. Revised approach avoids C++ entirely:

- **DSF**: lofty already supports DSD/DSF natively — just add the extension
- **WMA, MKA**: `mediainfo --Output=JSON` subprocess, same tool the Python anime
  module already uses

No new Cargo dependencies. No C++ cross-compilation complexity.

## Implementation

All changes in `etp-lib/src/metadata.rs`:

1. Custom `MetadataError` enum replacing `lofty::error::LoftyError`
2. Extension-based dispatch: `.wma`/`.mka` → mediainfo, everything else → lofty
3. `read_metadata_mediainfo` runs `mediainfo --Output=JSON`, parses properties
   from the Audio track, tags from the General track
4. `normalize_mediainfo_tag_key` maps mediainfo field names to canonical tag
   names consistent with lofty normalization
5. `AUDIO_EXTENSIONS` extended with `dsf`, `mka`, `wma`

## Limitations

- No embedded image extraction for WMA/MKA (mediainfo reports presence but can't
  extract bytes). Acceptable for the small number of files.
- mediainfo must be installed on the system. Clear error message if missing.
